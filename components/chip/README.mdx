## Usage

Concise representation of an otherwise more complex entity.

## Rendering Basic Chips

Chips contain a label (text) and can optionally render an avatar. The avatar can be an icon or any other renderable node.

<Playground data-test="basic-chips">
  <Chip label="basic text chip" />
  <Chip avatar={<span>i</span>} label="chip with avatar" />
  <Chip fullWidth label="full width chip" />
  <Chip fullWidth avatar={<span>i</span>} label="full width with avatar" />
</Playground>

## Using Custom Component

In some cases, you may want a Chip to actually be an anchor (link). Chip provides a way to supply a component to be used as its root.

> When using an anchor for a Chip, the avatar and chip become the anchor's clickable target. No click handler will be invoked; regardless of the `clickable` prop value.

<Playground data-test="custom-component">
  {() => {
    const AssetLink = ({ href, children }) => (
      <a href={href}>
        {children}
      </a>
    );

    return (
      <Fragment>
        <Chip
          avatar={<span>i</span>}
          component="a"
          href="https://verdigris.andrew.codes"
          label="a clickable link chip"
          onClick={console.log}
        />
        <Chip
          avatar={<span>i</span>}
          component={AssetLink}
          href="https://verdigris.andrew.codes"
          label="an AssetLink chip"
          onClick={console.log}
        />
      </Fragment>
    );

}}

</Playground>

## Clickable Chips

Chips can also be configured to be clickable. See the `console` for logged messages when clicking.

<Playground data-test="clickable-chips">
  <Chip clickable label="a clickable chip" onClick={console.log} />
  <Chip
    clickable
    avatar={<span>i</span>}
    label="a clickable chip with avatar"
    onClick={console.log}
  />
  <Chip
    clickable
    avatar={<span>i</span>}
    component="a"
    href="https://verdigris.andrew.codes/"
    label="a clickable link chip"
    onClick={console.log}
  />
</Playground>

## Deletable Chips

Finally, chips can be configured to contain a delete button and associated click handler. See the `console` for logged messages when deleting.

> Remember, chips can be both deletable **and** clickable at the same time. In these cases, only one event handler will be invoked per click.

<Playground data-test="deletable-chips">
  <Chip
    avatr={<span>i</span>}
    label="a deletable chip"
    onDelete={console.log}
  />
  <Chip
    avatar={<span>i</span>}
    label="a deletable chip with avatar"
    onDelete={console.log}
  />
  <Chip
    clickable
    avatar={<span>i</span>}
    label="a clickable, deletable chip"
    onClick={console.log}
    onDelete={console.log}
  />
</Playground>
